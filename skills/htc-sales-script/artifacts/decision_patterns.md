# Decision Patterns - HTC Sales Script

## Como Esta Skill Toma Decisões

### 1. Hierarquia de Frameworks

Quando há múltiplas abordagens possíveis:

| Situação | Framework Principal | Suporte |
|----------|---------------------|---------|
| Estrutura da call | CCDI (Hormozi) | The Stack (Brunson) para close |
| Apresentação de valor | The Stack (Brunson) | Value Equation (Hormozi) para justificar |
| Storytelling | Epiphany Bridge (Brunson) | CCDI Connect (Hormozi) para timing |
| Handling objections | Hormozi (concordar + reframe) | Feel-Felt-Found (tradicional) |
| Closing | Assumptive (Hormozi) | Alternative (tradicional) como backup |
| Follow-up | 7-Touch (integrado) | Humano + automação |

### 2. Decision Tree: Qual Fase do CCDI?

```
ONDE VOCÊ ESTÁ NA CALL?
│
├─→ Primeiros 10 minutos
│   └─→ CONNECT
│       → Foco: Rapport, contexto, conforto
│       → Perguntas abertas sobre negócio/vida
│       → Establish: "Não é call de vendas tradicional"
│
├─→ Minuto 10-25
│   └─→ CHALLENGE
│       → Foco: Quebrar crenças, criar gap
│       → Perguntas sobre abordagem atual
│       → Questionar: "Isso vai te levar lá?"
│
├─→ Minuto 25-45
│   └─→ DISCOVER
│       → Foco: Dor profunda, custo de inação
│       → 5 Levels of Why
│       → Visão do futuro (Before/After Grid)
│
├─→ Minuto 45-55
│   └─→ INTENSIFY
│       → Foco: Urgência, solução única
│       → Transferir autoridade (eu tenho o mapa)
│       → Criar tensão que só ação resolve
│
└─→ Minuto 55-60
    └─→ CLOSE
        → The Stack completo
        → Assumptive close
        → Handle últimas objeções
```

### 3. Decision Tree: Como Lidar com Objecão?

```
QUAL OBJEÇÃO VOCÊ OUVIU?
│
├─→ "Está caro" / "Não tenho dinheiro"
│   └─→ FRAMEWORK: Custo de Inação
│       → "Você está certo, é investimento significativo."
│       → Calcular quanto está perdendo por mês
│       → Comparar: investimento vs custo de não agir
│
├─→ "Preciso pensar"
│   └─→ FRAMEWORK: Urgência Lógica (Brunson) + Brutal Honesty (Hormozi)
│       → "O que especificamente você precisa pensar?"
│       → "O que vai mudar em alguns dias?"
│       → "Você está com dúvida real ou com medo?"
│
├─→ "Preciso falar com [pessoa]"
│   └─→ FRAMEWORK: Permission-Based + Script
│       → Validar: "Excelente que você respeite [pessoa]"
│       → Perguntar: "Como essas conversas normalmente terminam?"
│       → Oferecer: "Quer que eu te dê um script?"
│
├─→ "Não tenho tempo"
│   └─→ FRAMEWORK: Prioridade Reframe
│       → "Você não tem tempo, ou isso não é prioridade?"
│       → "Se isso fosse resolver [dor], você faria tempo?"
│       → "Quanto tempo você gasta com [atividade de baixo retorno]?"
│
├─→ "Vou fazer sozinho"
│   └─→ FRAMEWORK: Economia Falsa
│       → "Você já tentou sozinho antes? Como foi?"
│       → "Eu gastei [tempo/dinheiro] para descobrir isso."
│       → "Você quer pagar com tempo ou com dinheiro?"
│
└─→ "Não funciona para mim"
    └─→ FRAMEWORK: Responsabilidade + Suporte
        → "Por que você acha isso?"
        → "O que faltou nas outras vezes?"
        → "Aqui está o que é diferente no nosso suporte..."
```

### 4. Decision Tree: Quando Fechar?

```
SINAIS DE QUE ESTÁ PRONTO PARA FECHAR:
│
├─→ Verbais
│   → "Como funciona?"
│   → "O que está incluído?"
│   → "Quando começamos?"
│   → "Qual o preço mesmo?"
│
├─→ Tonais
│   → Voz mais suave/relaxed
│   → Silêncios confortáveis
│   → "Hmm..." de consideração
│
└─→ Comportamentais
    → Pegando papel/caneta
    → Olhando para cartão/carteira
    → Relaxamento postural

QUANDO NÃO FECHAR AINDA:
→ Pergunta sobre features (ainda racionalizando)
→ "E se..." (ainda cético)
→ Tensão alta (não criou rapport suficiente)
```

### 5. Decision Tree: Qual Técnica de Fechamento?

```
PERFIL DO PROSPECT?
│
├─→ Decidido (sinais claros de compra)
│   └─→ ASSUMPTIVE CLOSE
│       → "Qual cartão você quer usar?"
│       → "Você prefere começar segunda ou terça?"
│
├─→ Quase lá (uma pequena objeção)
│   └─→ SHARP ANGLE CLOSE
│       → "Se eu resolver [objeção], você entra?"
│
├─→ Comparando opções
│   └─→ ALTERNATIVE CLOSE
│       → "Opção A ou Opção B?"
│
├─→ Analítico (precisa ver tudo organizado)
│   └─→ SUMMARY CLOSE
│       → Recapitular valor → "Faz sentido?"
│
├─→ Hesitante (medo de decidir)
│   └─→ QUESTION CLOSE
│       → "Do 1 ao 10, quanto você quer isso?"
│       → "Por que não 10?"
│
└─→ Resistente (muitas objeções)
    └─→ TAKEAWAY CLOSE
        → "Talvez isso não seja para você agora..."
        → Forçar reconhecimento de desejo
```

### 6. Prioridade de Descoberta (5 Whys)

```
QUANDO DISCOVERING, IR NESSA ORDEM:

1. SITUAÇÃO ATUAL (Superficial)
   → "Qual seu faturamento?"
   → "Quantos clientes?"

2. DESAFIOS (Situacional)
   → "O que não está funcionando?"
   → "O que você já tentou?"

3. EMOCIONAL (Importante)
   → "Como isso te afeta pessoalmente?"
   → "Quando você acorda, o que pensa?"

4. EXISTENCIAL (Crítico)
   → "Se continuar assim por 1 ano?"
   → "Como isso afeta sua família?"

5. IDENTIDADE (Mais profundo)
   → "Quem você precisa se tornar?"
   → "O que isso diz sobre você?"

PARAR QUANDO:
→ Prospect emocional
→ Silêncio pesado
→ "Nunca pensei nisso assim..."
→ Visão clara do "after"
```

### 7. When to Use Silence

```
USE SILÊNCIO APÓS:

✓ Pergunta de fechamento
  → "Você quer que eu te leve para lá?"
  → [SILÊNCIO]

✓ Apresentar preço
  → "O investimento é R$ 25.000"
  → [SILÊNCIO]

✓ Responder objeção
  → "Entendo. E se eu te mostrasse..."
  → [SILÊNCIO]

✓ Prospect emocional
  → "...e é por isso que preciso mudar"
  → [SILÊNCIO - deixar processar]

NUNCA USE SILÊNCIO:
✗ Quando prospect está confuso
✗ Quando há tensão negativa
✗ Quando você não sabe o que dizer
✗ Para manipular (seja genuíno)
```
