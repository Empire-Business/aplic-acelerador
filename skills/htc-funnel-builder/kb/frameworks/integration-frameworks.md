# Frameworks de Integração Brunson + Hormozi

## O HTC Framework (High-Ticket Conversion)

Combinação máxima para vendas high-ticket de R$ 10.000 a R$ 100.000+.

---

## Framework 1: The Perfect Stack

**Combinação:** Brunson's "The Stack" + Hormozi's "Value Equation"

### Passo a Passo:

```
1. APRESENTAR COMPONENTES (The Stack)
   Cada item com valor atribuído:
   
   "Componente 1: [Nome] = R$ X.XXX
    Componente 2: [Nome] = R$ XXX
    Componente 3: [Nome] = R$ XXX"

2. SOMAR VALORES (Anchor High)
   
   "Valor total: R$ XX.XXX"

3. APLICAR VALUE EQUATION
   
   "Mas aqui está o que você realmente recebe:
   
   → Dream Outcome: [Resultado específico]
   → Likelihood: [X]% baseado em [prova]
   → Time: [Tempo curto] vs [tempo longo]
   → Effort: Sistema pronto vs fazer sozinho
   
   Valor real: R$ [cálculo Value Equation]"

4. REVELAR INVESTIMENTO
   
   "Seu investimento: R$ [Preço]
    
    ROI em [tempo]: [X]x"

5. ADICIONAR GARANTIA (Hormozi-style)
   
   "E você está 100% protegido com [garantia específica]"
```

## Framework 2: High-Ticket Application System

**Combinação:** Brunson's funnel structure + Hormozi's qualification

### Etapas:

```
FASE 1: PRÉ-QUALIFICAÇÃO (Brunson)
├── Conteúdo que educa e posiciona
├── VSL de 10-20 min (opcional)
└── Prova de autoridade

FASE 2: APLICAÇÃO (Hormozi-style)
├── Formulário detalhado
├── Score de qualificação (BANT)
└── Filtro de budget/timeline

FASE 3: DISCOVERY (Hormozi CCDI - Connect)
├── Rapport building
├── Entender situação atual
├── Identificar dor profunda
└── Estabelecer confiança

FASE 4: CHALLENGE (Hormozi CCDI - Challenge)
├── Questionar abordagem atual
├── Quebrar crenças limitantes
├── Criar GAP (onde está vs onde quer)
└── Posicionar como guia

FASE 5: SALES CALL (Brunson + Hormozi)
├── Recap do discovery (The Stack início)
├── Apresentar solução (3 Secrets light)
├── Valor via Value Equation
├── Remover objeções
├── The Stack completo
└── Assumptive close
```

## Framework 7: High-Ticket Closing (CCDI + The Stack)

**Script integrado para calls de vendas:**

```
PHASE 1: CONNECT (0-10 min) — Hormozi
"[Nome], esta não é uma call de vendas tradicional.
Vou entender onde você está, o que já tentou,
e juntos vamos descobrir se faz sentido trabalharmos."

PHASE 2: CHALLENGE (10-25 min) — Hormozi
"Deixa eu entender... você está fazendo [X],
isso gera [Y], e você espera chegar em [Z]?
Matematicamente, quanto tempo levaria?"

PHASE 3: DISCOVERY (25-45 min) — Hormozi
"Se nada mudar em 12 meses, onde você estará?
Como isso afeta sua família? Sua saúde?
Quanto está custando NÃO resolver isso?"

PHASE 4: INTENSIFY (45-55 min) — Hormozi + Brunson
"Você está na frente de uma parede invisível.
Eu tenho o mapa. Você quer seguir?"

PHASE 5: THE STACK (55-60 min) — Brunson
"Deixa eu mostrar exatamente o que você recebe...

[COMPONENTE 1] = R$ X
[COMPONENTE 2] = R$ X
...
Total: R$ XX

Hoje: R$ [Preço]

Qual cartão você quer usar?"
```

## Framework 8: Testing & Optimization System

**Combinação:** Hormozi's Test 10x + Brunson's metrics

```
PARA CADA ELEMENTO:

1. IDENTIFICAR (Brunson: 23 Building Blocks)
   → Qual bloco tem maior impacto potencial?

2. CRIAR VARIAÇÕES (Hormozi: 10x)
   → 10 headlines diferentes
   → 10 ofertas diferentes
   → 10 CTAs diferentes

3. TESTAR (Brunson: Split testing)
   → Dividir tráfego igualmente
   → Rodar até significância estatística
   → Mínimo 100 conversões por variação

4. ANALISAR (Hormozi: Data-driven)
   → Winner pega 70% do tráfego
   → Criar 10 novas variações do winner

5. ESCALAR (Brunson: Scale what works)
   → Aumentar budget no winner
   → Aplicar aprendizado a outros blocos

PRIORIDADE DE TESTE:
1. Hook/Headline (80% do impacto)
2. Oferta (preço, bônus, garantia)
3. CTA
4. Prova social
5. Design
```
