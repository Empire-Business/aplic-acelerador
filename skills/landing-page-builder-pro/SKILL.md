## Etapa 2: ANÁLISE E ESTRATÉGIA

### Framework de Diagnóstico (use SEMPRE)

Antes de escrever uma linha de código, aplique a **Value Equation do Hormozi**:

```
VALUE = (Dream Outcome × Perceived Likelihood of Success)
        ÷ (Time Delay × Effort & Sacrifice)
```

**Pergunte-se:**
1. **Dream Outcome**: Qual sonho a página está vendendo?
2. **Perceived Likelihood**: Como a página aumenta a credibilidade de que funciona?
3. **Time Delay**: Como a página promete resultado rápido?
4. **Effort & Sacrifice**: Como a página promete "caminho fácil"?

### Checklist de Grand Slam Offer (Hormozi)

```
□ O offer resolve TODOS os problemas? (não só alguns)
□ É incomparável? (não tem concorrente direto)
□ Cria discrepância de valor massiva? (valor >> preço)
□ Tem escassez/urgência REAIS? (não falsas)
□ Tem bônus que aceleram resultado?
□ Tem garantia que remove risco?
□ Tem nome memorável? (não genérico)
```

### Estrutura de Copy (Long-form DR)

```markdown
## HEADLINE (3 níveis)
- Principal: Benefício + Especificidade + Gancho
- Sub: Mecanismo + Prazo + Para quem
- Qualificação: Quem deve continuar lendo

## LEAD (escolha 1)
A) História de dor (conexão emocional)
B) Contraste/verdade incômoda (choque)
C) Mecanismo revelado (curiosidade)

## AGITAÇÃO DO PROBLEMA
- Mostre consequências práticas
- Ciclo de tentativas frustradas
- Inimigo comum (má orientação)

## A VIRADA
"O problema não é X... é Y."
Revelação do mecanismo único.

## PROVA EM CAMADAS
1. Credenciais/autoridade
2. Cases e números
3. Demonstração lógica
4. Depoimentos (se houver)

## APRESENTAÇÃO DO MÉTODO
- Passo a passo claro
- Simples o suficiente para entender
- Complexo o suficiente para valer

## OFERTA (STACK)
- Item 1: Nome + Benefício + Valor
- Item 2: Nome + Benefício + Valor
- Item 3: Nome + Benefício + Valor
- Bônus 1, 2, 3...
- TOTAL: R$ X
- HOJE: R$ Y

## GARANTIA
Risco reverso claro e simples.

## OBJEÇÕES DESTRUIDAS
- "Não tenho tempo" → Reenquadre
- "Não tenho dinheiro" → Custo da inação
- "Já tentei" → Por que era diferente
- "Não vai funcionar pra mim" → Prova específica

## CTA (repetido 6+ vezes)
Comando claro + O que acontece + Garantia

## FAQ
10-15 perguntas reais com respostas honestas

## FECHAMENTO
Urgência real + Escassez real + Convicção final
```

### Tom de Voz da Copy

**APLIQUE ESTE ESTILO:**

```
✅ FAÇA:
- Frases curtas e diretas
- Travessões para cortes mentais —
- Especificidade (números, prazos, nomes)
- Curiosidade em cada parágrafo
- Tensão que cresce até o CTA
- Empatia real com a dor
- Convicção absoluta

❌ NÃO FAÇA:
- Marketingês (sinergia, disruptivo, 360)
- Promessas vazias sem prova
- Urgência falsa
- Texto neutro ou consultivo
- Parágrafos longos
- Linguagem acadêmica
```

## Etapa 4: DESIGN DE NÍVEL MUNDIAL

### Princípios de Design (SEMPRE APLIQUE)

Antes de codar, defina a **DIREÇÃO ESTÉTICA**:

```
Escolha UM destes estilos:
□ BRUTALIST - Raw, bold, quebra de grid
□ MINIMALIST - Clean, whitespace, refinado
□ LUXURY - Premium, elegante, dourado
□ TECH - Dark mode, neon, futurista
□ ORGANIC - Natural, terracota, quente
□ EDITORIAL - Magazine, tipografia forte
□ PLAYFUL - Cores vibrantes, formas divertidas
□ CORPORATE - Profissional, confiável, azul
```

### Hierarquia Tipográfica

```css
/* NUNCA use fontes genéricas (Inter, Roboto, Arial) */

/* Headlines: Fonte display com personalidade */
--font-display: "Playfair Display" | "Bebas Neue" | "Oswald" | "Crimson Pro";

/* Body: Fonte legível mas não padrão */
--font-body: "Source Sans 3" | "Work Sans" | "DM Sans" | "IBM Plex Sans";

/* Escalas */
h1: clamp(2.5rem, 8vw, 5rem)
h2: clamp(1.75rem, 5vw, 3rem)
h3: clamp(1.25rem, 3vw, 1.75rem)
body: clamp(1rem, 2vw, 1.125rem)
```

### Sistema de Cores

```css
/* Paleta limitada com propósito */
:root {
  /* Cor principal de ação */
  --primary: #2563eb;       /* CTA, links */

  /* Cor de destaque */
  --accent: #f59e0b;        /* Highlights, badges */

  /* Cores neutras */
  --bg: #0a0a0a;            /* Fundo (dark mode first) */
  --surface: #141414;       /* Cards, boxes */
  --text: #ffffff;          /* Texto principal */
  --muted: #888888;         /* Texto secundário */
  --border: #262626;        /* Bordas */
}
```

### Espaçamento (8px Grid)

```css
/* Sistema de espaçamento consistente */
--space-xs: 0.5rem;   /* 8px */
--space-sm: 0.75rem;  /* 12px */
--space-md: 1rem;     /* 16px */
--space-lg: 1.5rem;   /* 24px */
--space-xl: 2rem;     /* 32px */
--space-2xl: 3rem;    /* 48px */
--space-3xl: 4rem;    /* 64px */
--space-4xl: 6rem;    /* 96px */
```

### Animações (com propósito)

```css
/* Micro-interações que valem a pena */

/* 1. Fade in up na entrada */

  from { opacity: 0; transform: translateY(20px); }

  opacity: 1;
  transform: translateY(0);
```

## Etapa 5: GERAÇÃO DE IMAGENS (IA)

### Quando Gerar vs Pedir

```
SEMPRE GERE quando:
□ Usuário não tem imagens
□ Precisa de hero image
□ Precisa de ilustrações de features
□ Precisa de avatares de depoimentos
□ Precisa de ícones customizados

PEÇA AO USUÁRIO quando:
□ É foto real do produto
□ É foto da pessoa/equipe
□ É screenshot de software
□ É logo da marca
□ É case/print real
```

## REGRAS FINAIS

1. **Nunca pule o intake** - Informação ruim = página ruim
2. **Sempre peça imagens** - Se não tiver, gere por IA
3. **Copy é prioridade** - Design bonito sem copy é perda de tempo
4. **Teste dos 5 segundos** - Se não entender em 5s, refaça
5. **Urgência real ou nenhuma** - Nunca use falsa urgência
6. **Entregue completo** - Código pronto para publicar
