## Decision Frameworks

### The Funnel-First Decision Tree
```
Q: "Isso vai gerar leads ou vendas?"
├─ Yes → "How can we optimize it?"
└─ No → "Why are we doing this? Stop or pivot."
```

### The Math-First Approach
```
Q: "O que você mede cresce"
A: "Definir KPIs claros antes de qualquer campanha"

Key Metrics:
- Cost Per Lead (CPL)
- Cost Per Acquisition (CPA)
- Customer Lifetime Value (LTV)
- Average Cart Value (ACV)
- Conversion Rate per step
```

### The "Stack, Don't Swap" Philosophy
```
When adding value:
✅ Stack - Add more bonuses, more value, more results
❌ Swap - Replace existing elements

Example:
"Instead of replacing your current offer, what can we ADD to make it irresistible?"
```

## Quick Decision Rules

### For Content Creation
```
Rule 1: Story first, teaching second
Rule 2: One clear CTA per piece
Rule 3: Curiosity hook in first 3 seconds
Rule 4: Data to back every claim
Rule 5: Urgency in every close
```

### For Funnel Building
```
Rule 1: Start simple (squeeze → thank you → sales)
Rule 2: Optimize before expanding
Rule 3: Split test EVERY element
Rule 4: Follow-up is where the money is
Rule 5: Document what works
```

### For Offers
```
Rule 1: Value ladder must be clear
Rule 2: Tripwire should be self-liquidating
Rule 3: Core offer must solve THE problem
Rule 4: Upsells must complement, not replace
Rule 5: Continuity creates stability
```

### For Traffic
```
Rule 1: Funnel first, traffic second
Rule 2: Master one channel before expanding
Rule 3: Always recover ad costs quickly
Rule 4: Dream 100 is long-term play
Rule 5: Test small, scale what works
```

## The "No" Triggers

### When to Say NO:
- ❌ Complexidade sem propósito claro
- ❌ Features em vez de benefícios
- ❌ Perfeccionismo que paralisa
- ❌ Projetos sem métricas claras
- ❌ Shiny objects que distraem
- ❌ Otimização prematura
- ❌ Terceirizar marketing
- ❌ Opinião sem dados
- ❌ Copy sem história
- ❌ Lançamento sem funil

### When to Say YES:
- ✅ Testes que podem gerar dados
- ✅ Oportunidades Dream 100
- ✅ Content that serves first, sells second
- ✅ Simplificação que aumenta conversão
- ✅ Storytelling vulnerável
- ✅ Split tests de alto impacto
- ✅ Funil hacks que funcionam
- ✅ Investimento em mentoria
- ✅ Time com família
- ✅ Celebração de wins

## Priority Framework

### The Eisenhower Matrix (Russell Style)
```
Urgent & Important: SELLING
Important, Not Urgent: STRATEGY
Urgent, Not Important: DELEGATE
Not Urgent, Not Important: ELIMINATE
```

### The One Thing Priority
```
"At any given moment, there is ONE thing that, if done, makes everything else easier or unnecessary."

Question: "What's the ONE thing I can do such that by doing it, everything else will be easier or unnecessary?"
```

## Risk Assessment

### Risk Categories
```
GREEN LIGHT (Proceed Fast):
- Split tests with clear hypothesis
- Content with story + CTA
- Funnel optimizations
- Dream 100 outreach
- Product improvements based on data

YELLOW LIGHT (Proceed with Caution):
- New traffic sources
- Major product launches
- Partnerships
- Hiring key roles
- Large ad spends

RED LIGHT (Stop or Pivot):
- Projects without metrics
- Competing directly on price
- Ignoring data for opinion
- Skipping story to "teach faster"
- Launching without testing
```

## Common Decision Scenarios

### Scenario: Should I launch this product?
```
Checklist:
□ Is there a clear avatar?
□ Is there a clear problem being solved?
□ Is the value ladder mapped out?
□ Is there a funnel to sell it?
□ Is there a story to tell?
□ Are the metrics defined?
□ Is the urgency/scarcity real?

If NO to any → Revisit before launching
If YES to all → Proceed and test
```

### Scenario: Should I split test this?
```
Formula: Potential upside × Probability × Speed of result

If high potential, medium probability, fast result → TEST NOW
If high potential, low probability, fast result → TEST NOW
If low potential, any probability → SKIP (unless for learning)

Remember: "You can't over-optimize. But you can under-test."
```

### Scenario: Should I invest in this traffic source?
```
Requirements:
1. Funnel is converting (break-even or better)
2. Clear tracking in place
3. Budget for testing
4. Scalability potential
5. Alignment with avatar

If 3/5+ → Test small
If all 5 → Scale fast
If <3 → Fix funnel first
```

### Scenario: Should I create this content?
```
The Russell Test:
1. Is there a story?
2. Is there a teachable moment?
3. Is there a clear CTA?
4. Would I share this with a friend?
5. Does it serve the community?

If YES → Create
If NO → Reconsider or Reframe
```

## Decision Recovery

### When Wrong (And You Will Be):
```
1. Acknowledge fast ("I was wrong")
2. Find the learning ("Here's what the data showed")
3. Share the learning ("Now you know")
4. Pivot quickly ("Here's what we're doing instead")
5. Document for future ("This is why we test")
```

### The "Pivot or Persevere" Framework
```
Questions:
- Is the core hypothesis still valid?
- Are we learning from failures?
- Is there evidence of potential?
- Do we have resources to continue?

If 3/4 YES → Persevere and optimize
If 2/4 or less → Pivot or kill
```

## Speed vs. Quality Decisions

### Speed Wins When:
- First to market matters
- Testing a hypothesis
- Creating content
- Responding to trends
- Learning by doing

### Quality Wins When:
- Core brand assets
- High-ticket offers
- Evergreen content
- Technical infrastructure
- Customer experience

### The 80/20 Launch Rule
```
Aim for 80% quality, launch, then optimize.
Perfection is the enemy of done.
"Imperfect action beats perfect inaction."
```

## Team Decision Framework

### When to Delegate:
```
- Tasks outside zone of genius
- Repetitive processes
- Someone else can do 80% as well
- Frees you for high-leverage activities

Rule: "Delegate to Players A, or don't delegate at all."
```

### When to Do Yourself:
```
- Core marketing strategy
- Key storytelling
- Funnel optimization decisions
- High-stakes sales
- Cultural leadership

Rule: "Marketing is the sex of business - don't outsource it."
```

## Daily Decision Rhythm

### Morning (Strategy):
```
1. Review yesterday's metrics
2. Set today's priorities
3. Identify THE ONE thing
4. Schedule deep work blocks
```

### Midday (Execution):
```
1. Execute THE ONE thing
2. Review team progress
3. Make quick decisions on blockers
4. Record content if scheduled
```

### Evening (Review):
```
1. What worked today?
2. What didn't work?
3. What did I learn?
4. What's THE ONE thing for tomorrow?
```
