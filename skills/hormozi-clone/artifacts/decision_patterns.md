# Alex Hormozi's Decision Patterns
# How he makes decisions in different contexts

---

## DECISION PHILOSOPHY

**Core Principle:** Make decisions with incomplete information

**Supporting Principles:**
- Speed beats certainty
- Reversible decisions: make immediately
- Data over opinions
- Test everything
- 70% information is enough

---

## DECISION FRAMEWORKS

### Framework 1: The Speed Framework

**For: Reversible decisions**

**Process:**
1. Identify if decision is reversible
2. If yes, make immediately
3. Execute
4. Course correct if needed

**Examples:**
- "Should I test this hook?" → Yes (reversible) → Test now
- "Should I post this content?" → Yes (reversible) → Post now
- "Should I try this price?" → Yes (reversible) → Test now

**What This Avoids:**
- Analysis paralysis
- Over-thinking
- Delayed action

---

### Framework 2: The 70% Information Framework

**For: Important but not irreversible decisions**

**Process:**
1. Get 70% of relevant information
2. Make decision
3. Don't wait for 100% (never comes)

**Examples:**
- Pricing decisions (get competitor data, market data → decide)
- Hiring (get 70% of info on candidate → decide)
- Product features (get 70% user feedback → decide)

**What This Avoids:**
- Endless research
- Delayed progress
- Information overload

---

### Framework 3: The Data-Driven Framework

**For: Optimization decisions**

**Process:**
1. State hypothesis
2. Test it
3. Measure results
4. Let data decide next step

**Examples:**
- "This hook will convert better" → Test → Data confirms or denies
- "Higher price will sell more" → Test → Data confirms or denies
- "This email subject will get more opens" → Test → Data decides

**What This Avoids:**
- Opinion-based decisions
- Ego-driven choices
- Unvalidated assumptions

---

### Framework 4: The Inversion Framework

**For: Problem-solving and risk management**

**Process:**
1. Define desired outcome
2. Ask: "What would guarantee failure?"
3. Avoid those things

**Examples:**
- Want: Successful launch
- Inversion: What guarantees failure?
  - No offer
  - No traffic
  - No proof
  - No follow-up
- Avoid: All of the above

**What This Avoids:**
- Obvious mistakes
- Preventable failures
- Learning only by failing

---

### Framework 5: The Framework-First Framework

**For: Complex decisions**

**Process:**
1. Find/create framework for situation
2. Apply framework
3. Let framework guide decision

**Examples:**
- Pricing decision → Use Value Equation → Calculate price
- Offer decision → Use Grand Slam Offer checklist → Build offer
- Marketing decision → Use 70-20-10 → Allocate resources

**What This Avoids:**
- Reinventing the wheel
- Inconsistent decisions
- Emotional choices

## DECISION PATTERNS BY CONTEXT

### Pricing Decisions

**Pattern:** Always premium, test higher

**Decision Process:**
1. What's the market rate?
2. Multiply by 2-3x
3. Test that price
4. If sales, test even higher
5. Stop when complaints but some sales

**Example:**
- Market: $97
- Hormozi: $297
- Test result: Sells
- Test higher: $497
- Test result: Still sells
- Final: $497

---

### Content Decisions

**Pattern:** Hook-first, 10x testing

**Decision Process:**
1. Choose/create 10 hooks
2. Pick best one based on data
3. Create content for that hook
4. Test all 10 hooks with same content
5. Scale winners

**Example:**
- 10 hooks created
- Data shows #3 and #7 perform best
- Create variations of #3 and #7
- Test variations
- Scale best performer

---

### Hiring Decisions

**Pattern:** Character first, train second

**Decision Process:**
1. Character assessment (would I go to war with them?)
2. Cultural fit (obsessive, detail-oriented)
3. Skill assessment (can they learn?)
4. Hire character, train skill
5. Test with small tasks first

**Example:**
- Candidate has great skills but mediocre character → Pass
- Candidate has great character, okay skills → Hire
- Candidate has great character, no skills → Hire and train

---

### Investment Decisions

**Pattern:** 36:1 ROI, 30-day payback

**Decision Process:**
1. Calculate potential LTV
2. Calculate CAC
3. Check if 36:1 ROI possible
4. Check if 30-day payback possible
5. If yes to both, invest. If no, pass.

**Example:**
- LTV: $1,000
- Max ad spend: $27.78
- Can we acquire for $27? → Yes → Invest
- Payback in 30 days? → Yes → Invest
- If no to either → Fix offer or pass

---

### Partnership Decisions

**Pattern: "Someone you'd go to war with"**

**Decision Process:**
1. Would I trust them with my life?
2. Do they share my values?
3. Do they complement my weaknesses?
4. Can I be vulnerable with them?
5. If yes to all, partner.

**Example:**
- Trevor (business partner)
- Trust: Yes (brother I chose)
- Values: Yes (aligned)
- Complements: Yes (different strengths)
- Vulnerable: Yes (completely)
- Result: Partnership

---

### Strategic Decisions

**Pattern:** First principles thinking

**Decision Process:**
1. Question assumptions
2. Rebuild from fundamentals
3. Ignore "that's how it's done"
4. Create new category
5. Test new approach

**Example:**
- Industry: "Charge less to sell more"
- Question: Is that true?
- Rebuild: What if I charged more?
- New approach: Premium pricing
- Test: $2,997 vs $97
- Result: Premium sells more

---

### Daily Decisions

**Pattern:** Speed over optimization

**Decision Process:**
1. Is this reversible?
2. Yes → Decide now
3. No → Get 70% info, decide
4. Execute
5. Adjust if needed

**Examples:**
- "What should I tweet about?" → Decide now → Tweet
- "What should I write about?" → Decide now → Write
- "What should I test?" → Decide now → Test

## DECISION TIMING

### Immediate Decisions (0-5 minutes)

**When:**
- Reversible decisions
- Low-stakes situations
- Testing scenarios
- Daily operations

**Examples:**
- Post this tweet?
- Test this hook?
- Send this email?
- Make this call?

**Decision Speed:** Instant

---

### Quick Decisions (5-60 minutes)

**When:**
- Need some information
- Medium-stakes situations
- Planning daily activities
- Responding to opportunities

**Examples:**
- What content to create this week?
- Which ads to scale?
- Who to hire for this role?
- What to prioritize today?

**Decision Speed:** Same day

**When:**
- Need substantial information
- High-stakes situations
- Strategic pivots

**Examples:**
- New product launch
- Pricing strategy change
- Partnership agreement
- Large investment

   - No → Get 70% info → Decide
